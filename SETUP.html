🚀 意識チェックシート - Firebase & GitHub Pages セットアップガイド

✅ ようこそ！ このガイドでは、アプリケーションを Firebase とGitHub Pages に統合する手順を説明します。

ステップ1️⃣：Firebase プロジェクトの作成

まず、Firebase Console でプロジェクトを作成します。

  1. Firebase Console にアクセス
  2. Googleアカウントでログイン
  3. 「プロジェクトを作成」をクリック
  4. プロジェクト名を入力（例：awareness-check-sheet）
  5. 「続行」をクリック
  6. Google Analytics は「有効化」を選択（推奨）
  7. 「プロジェクトを作成」をクリック
💡 ヒント: プロジェクト ID は後で使用するので、メモしておきましょう。例：awareness-check-sheet-abc123

ステップ2️⃣：Firebase 設定を取得

プロジェクト設定から、アプリケーションの設定を取得します。

  1. Firebase Console でプロジェクトを開く
  2. 左上のプロジェクト設定アイコン（⚙️）をクリック
  3. 「プロジェクト設定」タブを選択
  4. 下にスクロールして「マイアプリ」を探す
  5. 「ウェブ」アイコンをクリック（初回の場合は「アプリを追加」から）
  6. アプリのニックネームを入力（例：awareness-app）
  7. 「アプリを登録」をクリック
  8. 表示された設定をコピー

設定例：

const firebaseConfig = { apiKey: "AIza...", authDomain: "awareness-check-sheet-abc123.firebaseapp.com", projectId: "awareness-check-sheet-abc123", storageBucket: "awareness-check-sheet-abc123.appspot.com", messagingSenderId: "123456789000", appId: "1:123456789000:web:abc123def456" };
⚠️ 重要: apiKey はGitHubに公開される可能性があります。Firebaseのセキュリティルールで保護しましょう。

ステップ3️⃣：src/firebase-config.js を編集

ダウンロードした設定をファイルに貼り付けます。

  1. src/firebase-config.js をエディタで開く
  2. 以下の部分を見つける：
    export const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT_ID.firebaseapp.com", // ... };
  3. ステップ2で取得した設定で置き換える
  4. ファイルを保存
✅ 完了！ Firebase 設定がアプリに反映されました。

ステップ4️⃣：ローカルで動作確認

設定が正しく機能しているか確認します。

  1. ターミナル / コマンドプロンプトを開く
  2. プロジェクトディレクトリに移動：
    cd "c:\My For7\スーパーノート-アプリ開発\個人チェック表の開発"
  3. 依存関係をインストール：
    npm install
  4. ローカルサーバーを起動：
    npm run dev
  5. ブラウザで http://localhost:8080 を開く
  6. ブラウザのコンソール（F12キー）を開き、エラーがないか確認
✅ 成功！ アプリが正常に動作している場合、以下のメッセージが表示されます：
✅ Firebase 初期化完了

ステップ5️⃣：GitHub リポジトリを作成

  1. GitHub で新しいリポジトリを作成
  2. リポジトリ名：awareness-check-sheet
  3. 説明：毎日の意識を記録するチェックシート - Firebase & GitHub Pages版
  4. 「Public」を選択
  5. 「Add .gitignore」はスキップ（既に .gitignore があります）
  6. 「Create repository」をクリック
💡 ヒント: リポジトリURLは https://github.com/your-username/awareness-check-sheet のようになります。

ステップ6️⃣：Git リポジトリを初期化

ローカルコードを GitHub に同期します。

  1. ターミナルでプロジェクトディレクトリにいることを確認
  2. Git を初期化（初回のみ）：
    git config --global user.name "Your Name" git config --global user.email "your-email@example.com"
  3. ファイルをステージ：
    git add .
  4. 初期コミット：
    git commit -m "Initial commit: Firebase & GitHub Pages setup"
  5. リモートリポジトリを追加：
    git remote add origin https://github.com/your-username/awareness-check-sheet.git
  6. main ブランチに push：
    git branch -M main git push -u origin main
✅ 完了！ コードが GitHub にアップロードされました。

ステップ7️⃣：GitHub Pages を有効化

  1. GitHub リポジトリを開く
  2. 「⚙️ Settings」をクリック
  3. 左メニューの「Pages」をクリック
  4. 「Build and deployment」の「Source」で「GitHub Actions」を選択
  5. 「Save」をクリック
💡 ヒント: GitHub Actionsにより、コード変更時に自動的にデプロイされます。

ステップ8️⃣：Firebase 認証を有効化

後で認証機能を実装するため、事前に有効化します。

  1. Firebase Console でプロジェクトを開く
  2. 左メニューの「Authentication」をクリック
  3. 「Sign-in method」タブを開く
  4. 「Google」をクリック
  5. 「有効にする」をオンにする
  6. プロジェクト名を入力（表示用）
  7. 「保存」をクリック
  8. 同様に「Email/Password」も有効化（推奨）
✅ 完了！ 認証の準備が整いました。

ステップ9️⃣：Realtime Database を作成

  1. Firebase Console でプロジェクトを開く
  2. 左メニューの「Realtime Database」をクリック
  3. 「データベースを作成」をクリック
  4. ロケーションを選択（日本の場合 asia-northeast1）
  5. セキュリティルール：「テストモード」を選択
    ⚠️ 注意: テストモード（30日間制限）またはルール編集を選択。本番環境では必ずセキュリティルールを設定してください。
  6. 「有効にする」をクリック
✅ 完了！ Realtime Database が作成されました。

ステップ🔟：Cloud Storage を設定

  1. Firebase Console でプロジェクトを開く
  2. 左メニューの「Storage」をクリック
  3. 「バケットを作成」をクリック
  4. デフォルト設定のまま進める
  5. ロケーション：アジア（asia-northeast1）
  6. 「作成」をクリック
✅ 完了！ Cloud Storage が準備できました。

ステップ1️⃣1️⃣：デプロイ確認

  1. GitHub リポジトリを開く
  2. 「Actions」タブをクリック
  3. 「Firebase Deploy」ワークフローが実行中か確認
  4. 実行が完了するまで待機（通常3-5分）
  5. 実行が成功したら、「Deployments」で確認

🔧 トラブルシューティング

❌ Firebase に接続できない

  • ✓ src/firebase-config.js の設定を確認
  • ✓ Firebase Console でプロジェクトが有効になっているか確認
  • ✓ ブラウザのコンソール（F12キー）でエラーを確認
  • ✓ インターネット接続を確認

❌ GitHub Pages にアクセスできない

  • ✓ Settings → Pages で「GitHub Actions」が選択されているか確認
  • ✓ Actions タブでデプロイが成功しているか確認
  • ✓ URL が正しいか確認（https://your-username.github.io/awareness-check-sheet/）
  • ✓ ブラウザのキャッシュをクリア（Ctrl+Shift+Delete）

❌ 画像エクスポートが失敗する

  • ✓ 外部リソース（フォント等）が CORS で許可されているか確認
  • ✓ html2canvas ライブラリがロードされているか確認
